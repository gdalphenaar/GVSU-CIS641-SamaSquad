software requirements specifications

Overview

This document outlines the system requirements specification, consisting of functional and non-functional requirements, for Project Sama - our guitar temperature and humidity monitoring system.

This system consists of three major components: - the remote sensor(s) that measure local temperature and humidity; - the base unit that controls the remote sensor(s), analyzes and stores sensor data, and serves the web interface; and - the web interface that allows users to monitor environmental conditions

Functional and non-functional requirements will be listed for each major component.

Functional Requirements

  1. Remote Sensor(s)

    1. Sensor(s) shall be capable of taking and recording temperature readings.
    2. Sensor(s) shall be capable of taking and recording humidity readings.
    3. Sensor(s) shall transmit readings back to the base unit in a usable data form.
    4. Sensor(s) shall be capable of remote management by the base unit.

  2. Base Unit

    1. The base unit shall manage sensor(s) wirelessly.
    2. The base unit shall host a database to store sensor logs.
    3. The base unit shall host the server for the web interface.
    4. The base unit shall posess data analysis capabilities.
    5. The base unit shall be capable of continuous operation without external user intervention.

  3. Web Interface

    1. Users shall be able to access the web interface from a web browser on their local network.
    2. The web interface shall use data stored in the base unit's database (see functional requirement 2.2).
    3. Users shall be able to set upper and lower bounds (in degrees Fahrenheit or Celsius) for acceptable temperatures ranges.
    4. Users shall be able to set upper and lower bounds (in percent relative humidity) for acceptable humidity ranges.
    5. The web interface shall send a notification if either temperature or humidity drift out of the established acceptable range (see functional requirements 3.3 and 3.4).
    6. Users shall be able to specify a "binning" period for averaged readings.
    7. The web interface shall display the most recent averaged temperature reading.
    8. The web interface shall display the most recent averaged humidity reading.
    9. Users shall be able to specify a time period for trends.
    10. The web interface shall display temperature trends.
    11. The web interface shall display humidity trends.

Non-Functional Requirements

  1. Remote Sensor(s)

    1. Sensor(s) should accurately and reliably record and transmit temperature and humidity readings.
    2. Sensor(s) should operate in a power-efficient manner, and be capable of running off of rechargeable batteries.
    3. Sensor(s) should be able to be mounted to a guitar stand or stored in a guitar case.

  2. Base Unit

    1. The system should take averaged temperature and humidity readings over a time period specified by the users to reduce fluctuation (see functional requirement 3.6).
    2. All systems (i.e., dashboard server, control program, and analysis/logging routines) should be able to run on at minimum a Raspberry Pi 3 Model B.
    3. All systems (i.e., dashboard server, control program, and analysis/logging routines) should be able to run concurrently with minimal slowdown perceived by the user.
    4. The base unit should be able to restart all services after a reboot without significant user intervention (see functional requirement 2.5).
    5. Users should be able to install all necessary files to install and run this service through a simple command line script.

  3. Web Interface

    1. The web interface should be responsive and usable on a phone screen and/or other touch screen devices (while connected to the user's local network).
    2. Users should be able to access temperature and humidity logs, and either display them graphically or access them as data files.
    3. Users should be able to set up password-authenticated access to the web interface.
